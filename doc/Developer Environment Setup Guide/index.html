Developers Environment Setup Guide

1. SCM Setup

The Project is distributed via Git and hosted by GitHub.
=> GitHub Homepage: https://github.com/

I recommend using the official GitHub Client for Windows or Mac, which can be found on the GitHub Homepage. Go to the clients options and set the directory for your local GitHub projects as you like. To clone the repository to your local GitHub client, just open the projects URL and drag and drop the URL from your browser into the GitHub client.
To commit changes, you need a GitHub account. This account can get permission to commit changes by the project owner (me).



2. IDE Setup

As this project is an Apache Maven web application written in Java, so your IDE should support Maven. I recommend using the Eclipse IDE for Java EE Developers, which an be found at www.eclipse.org/downloads/. It includes maven integration via m2e by default.

To import the project into Eclipse, choose: File -> Import -> Maven -> Existing maven projects -> Next

Select the folder of your local git project repository that your created in the SCM Setup. If you used the GitHub official client, the path to the project folder can be found in the clients options. Make sure the pom.xml of the project is selected and hit Finish. The project should now appear in your project explorer.



3. Run the project

Make sure Eclipse has access to the internet. To test that, you can access the Eclipse Marketplace in Eclipse via Help -> Eclipse Marketplace

Rightclick on the eclipse project and hit Maven -> Update Project...
Make sure Offline is NOT selected, but everything else is selected. Then hit OK.You should do this whenever the pom.xml changes.

Rightclick on the eclipse project and hit Run As -> Maven Build...
Type tomcat7:run and select the Skip Tests option. Hit Apply and Run. You should now see the console logging output from the embedded tomcat server starting up and initializing the project servlet.

Open your browser and type localhost:8080/GMM in the browsers adress bar. You should now see the applications login screen.
